## Aim:
To perform a heuristic evaluation of existing mobile shopping applications, compare their UI/UX design principles, analyze usability issues, and redesign an improved interface using Figma.

## Algorithm:
```
Step 1: Select Applications
Chose 2–3 shopping applications/websites for UX comparison.
Example: Myntra, Amazon, and Ajio.
Step 2: Analyze Existing UI/UX
Observed navigation flow, color scheme, typography, layout, and usability.
Identified strengths and weaknesses in each application.
Step 3: Apply Heuristic Evaluation
Evaluated apps using UI/UX principles such as:
Consistency
Visibility
Feedback
Simplicity
Navigation efficiency
Step 4: Identify Problems
Noted usability issues like cluttered layouts, inconsistent spacing, or difficult navigation.
Step 5: Design Improved Interface in Figma
Created wireframes and high-fidelity mockups in Figma.
Designed splash screen and home screen for the BuyBuddy fashion app.
Used Auto Layout, components, icons, typography, and color hierarchy.
Step 6: Prototype Development
Linked screens using Figma Prototype mode.
Added interactive navigation and transitions.
Step 7: Final Evaluation
Compared redesigned interface with existing applications.
Verified improved usability, accessibility, and visual hierarchy.
Step 8: Export & Documentation
Exported final screens and presentation assets.
Uploaded project files to GitHub repository.
Generated lab report PDF for submission.
```
